Life Should be Easier...

And it can be. The Peripheral-Framework aims to simplify and streamline the proccess of creating peripherals for Minecraft mods such as ComputerCraft and OpenComputersComing Soon!. By acting as a buffer between unnecessarily complex APIs and your code, this framework will give you the freedom to write code that is consice, maintainable, and beautiful.

Download it Now

Why?

What were the motivations behind making this framework, and how badly was it needed? That's a good question.

The issue with the native ComputerCraft API is that writing peripherals that are maintainable, readable, and beautiful is nearly impossible without considerable work. As it is, developers are forced to produce code that is both verbose and unintuitive. To see exactly what I mean by this, head over to the comparison page and take a look at just how much the Peripheral-Framework improves your peripheral code.

How?

The Peripheral-Framework makes use of several Java annotations in order to interface between your code and the native mod APIs. In total, there are only 6 of these annotations that you need to worry about: and even then not all of them are required. If you want to see just how much this framework can do for you, go ahead and check out the documentation.
